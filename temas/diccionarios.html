1. Diccionarios: El Coraz√≥n del Mapeo de Datos en Python

En el vasto universo de la programaci√≥n, a menudo necesitamos asociar informaci√≥n de una manera que no sea simplemente secuencial. Queremos poder buscar un dato espec√≠fico utilizando un identificador √∫nico, tal como buscar√≠as una palabra en un diccionario para encontrar su significado, o el nombre de un elemento en la tabla peri√≥dica para conocer sus propiedades. Para esta tarea, Python nos ofrece una estructura de datos excepcionalmente poderosa y eficiente: el diccionario (dict). [cite: 11, 12]

Un diccionario es una colecci√≥n que almacena sus elementos en pares de clave-valor. Cada clave es √∫nica y sirve como una "etiqueta" o "√≠ndice" para acceder a su valor correspondiente. [cite: 14, 15]

El Modelo Clave-Valor: La Esencia del Diccionario

Entender el modelo clave-valor es fundamental para dominar los diccionarios[cite: 63]:

  • üîë Clave (Key): Es el identificador √∫nico para un valor dentro del diccionario. Piensa en ella como la palabra que buscas en un diccionario de idiomas, el s√≠mbolo de un elemento qu√≠mico, o el DNI de una persona.
    • Debe ser √∫nica; no puede haber claves repetidas.
    • Debe ser de un tipo de dato inmutable (ej: cadenas de texto, n√∫meros, tuplas que solo contengan elementos inmutables). [cite: 18, 64, 206] Las listas o otros diccionarios no pueden ser claves.
  • üè∑Ô∏è Valor (Value): Es la informaci√≥n o el dato asociado a una clave espec√≠fica. Puede ser la definici√≥n de una palabra, las propiedades de un elemento qu√≠mico, o los datos completos de una persona.
    • Puede ser de cualquier tipo de dato en Python: un n√∫mero, una cadena, una lista, otra tupla, ¬°incluso otro diccionario! [cite: 19, 66]
    • Los valores no necesitan ser √∫nicos; diferentes claves pueden apuntar al mismo valor.
  • üîó Par Clave-Valor (√çtem o Entrada): La combinaci√≥n clave: valor forma una "entrada" o "√≠tem" en el diccionario. [cite: 67, 176]

Por ejemplo, en un diccionario que almacena las masas at√≥micas de elementos qu√≠micos: la clave podr√≠a ser el s√≠mbolo del elemento (ej: "He" para Helio) y el valor ser√≠a su masa at√≥mica (ej: 4.002602).

Caracter√≠sticas Fundamentales de los Diccionarios:

  • Mutables: Puedes cambiar su contenido (a√±adir, modificar o eliminar pares clave-valor) despu√©s de crearlos. [cite: 20]
  • Ordenados (desde Python 3.7+): Mantienen el orden en que se insertaron los elementos. [cite: 22, 97, 213] En versiones anteriores, el orden no estaba garantizado.
  • Acceso R√°pido por Clave: La b√∫squeda, inserci√≥n y eliminaci√≥n de elementos usando su clave es, en promedio, muy r√°pida (tiempo constante, $O(1)$ [cite: 17, 32, 35]), gracias a su implementaci√≥n interna con tablas hash.
  • No permiten claves duplicadas: Si intentas a√±adir una clave que ya existe, el valor asociado a esa clave simplemente se actualizar√°. [cite: 18, 32]

üí° ¬øPor qu√© $O(1)$ es importante? [cite: 34, 36] La notaci√≥n $O(1)$ (le√≠da como "Orden de 1" o "tiempo constante") significa que la operaci√≥n toma aproximadamente la misma cantidad de tiempo, sin importar cu√°n grande sea el diccionario. Ya sea que tengas 10 √≠tems o 10 millones, encontrar un valor por su clave ser√° incre√≠blemente r√°pido. Esto contrasta con las listas, donde buscar un elemento puede requerir revisar cada √≠tem (operaci√≥n $O(n)$).

2. Creando Diccionarios en Python

Python te da varias maneras flexibles de crear diccionarios:

2.1. Con Llaves { } (Forma Literal):

Es la forma m√°s com√∫n y visual. Los pares clave: valor se separan por comas, todo encerrado en llaves.

    "Hierro": 7.874,
    "Cobre": 8.96,
    "Aluminio": 2.70,
    "Oro": 19.32

# Diccionario para representar un vector en 2D (Matem√°ticas)

print(f"Vector: {vector_2d}")

# Diccionario vac√≠o con llaves
dic_vacio1 = {}
print(f"Diccionario vac√≠o con {{}}: {dic_vacio1}, Tipo: {type(dic_vacio1)}")

2.2. Con la Funci√≥n dict():

El constructor dict() es muy vers√°til:

  • dict(): Crea un diccionario vac√≠o.
  • dict(clave1=valor1, clave2=valor2, ...): Usando argumentos de palabra clave (las claves deben ser nombres de identificador v√°lidos).
  • dict(iterable_de_pares): A partir de un iterable que produce pares clave-valor (ej: una lista de tuplas [('a', 1), ('b', 2)]).
  • dict(otro_diccionario): Crea una copia superficial de otro diccionario.
# Con argumentos de palabra clave (para datos de una simulaci√≥n f√≠sica)
simulacion_params = dict(
    tiempo_total_s=100, 
    paso_integracion_s=0.01, 
    metodo_integracion="Runge-Kutta"
)
print(f"Par√°metros de simulaci√≥n: {simulacion_params}")

# Desde una lista de tuplas (ej. notas musicales y sus frecuencias en Hz - M√∫sica)
notas_frecuencias_lista = [("La4", 440.00), ("Do5", 523.25), ("Sol3", 196.00)]
frecuencias_afinacion = dict(notas_frecuencias_lista)
print(f"Frecuencias de afinaci√≥n: {frecuencias_afinacion}")

# Diccionario vac√≠o con dict()

print(f"Otro vac√≠o con dict(): {dic_vacio2}")

2.3. Con el M√©todo de Clase dict.fromkeys(iterable, valor_opcional):

Crea un nuevo diccionario donde las claves se toman de un iterable y todas las claves apuntan inicialmente al mismo valor_opcional (que por defecto es None si no se especifica). 

# Para inicializar el inventario de espec√≠menes en un estudio biol√≥gico

# Salida: {'PlantaA': 0, 'InsectoB': 0, 'HongoC': 0}

# Para registrar la asistencia de alumnos a una clase (valor inicial False)

asistencia_clase = dict.fromkeys(lista_alumnos_clase, False)
print(f"Registro de asistencia inicial: {asistencia_clase}")

3. Acceder, Modificar y Eliminar Elementos

3.1. Accediendo a Valores por Clave

La principal forma de usar un diccionario es obtener el valor asociado a una clave.

A. Usando Corchetes [clave] (Acceso Directo):

Si la clave existe, obtienes su valor. Si la clave no existe, Python lanza un error KeyError. [cite: 75, 181] 

# Datos de un experimento de √ìptica (F√≠sica)
experimento_optica = {
    "fuente_luz": "L√°ser He-Ne",
    "longitud_onda_nm": 632.8,
    "potencia_mW": 5.0,
    "angulo_incidencia_grados": 30
}
longitud_onda = experimento_optica["longitud_onda_nm"]
print(f"Longitud de onda utilizada: {longitud_onda} nm")

# print(experimento_optica["polarizacion"]) # Esto dar√≠a KeyError

B. Usando el M√©todo get(clave, valor_por_defecto_opcional):

Es una forma m√°s segura si no est√°s seguro de que una clave exista. No lanza KeyError. [cite: 76, 110, 188]

  • Si la clave existe, devuelve su valor.
  • Si la clave no existe, devuelve None (por defecto).
  • Si la clave no existe y proporcionas un valor_por_defecto_opcional, devuelve ese valor.

# Inventario de instrumentos musicales (M√∫sica)
instrumentos_stock = {"Guitarra": 10, "Piano": 3, "Bater√≠a": 5}

stock_guitarra = instrumentos_stock.get("Guitarra")
print(f"Stock de Guitarras: {stock_guitarra}") # Salida: 10

stock_violin = instrumentos_stock.get("Viol√≠n", 0) # Si no hay violines, asumimos 0
print(f"Stock de Violines: {stock_violin}")   # Salida: 0

stock_bajo = instrumentos_stock.get("Bajo") # No existe, devuelve None
print(f"Stock de Bajos: {stock_bajo}")       # Salida: None

C. Usando el M√©todo setdefault(clave, valor_por_defecto_opcional):

Este m√©todo es interesante: si la clave existe, devuelve su valor (como get()). Pero si la clave no existe, la inserta en el diccionario con el valor_por_defecto_opcional (o None si no se especifica el valor por defecto) y luego devuelve ese valor por defecto. [cite: 110, 122, 193] 

# Conteo de frecuencia de palabras en un texto (Ling√º√≠stica/Procesamiento de Texto)
conteo_palabras = {}
texto_ejemplo = "la casa es azul y la ventana es azul"
palabras = texto_ejemplo.lower().split()

for palabra in palabras:
    # Si la palabra no est√°, la a√±ade con 0 y devuelve 0. Luego se suma 1.
    # Si ya est√°, devuelve su conteo actual. Luego se suma 1.
    conteo_palabras[palabra] = conteo_palabras.setdefault(palabra, 0) + 1

print(f"Frecuencia de palabras: {conteo_palabras}")
# Salida: {'la': 2, 'casa': 1, 'es': 2, 'azul': 2, 'y': 1, 'ventana': 1}

Puedes a√±adir un nuevo par o modificar el valor de una clave existente usando el operador de asignaci√≥n = con la clave entre corchetes. [cite: 20, 81, 182]

Tambi√©n puedes usar el m√©todo update(otro_diccionario_o_iterable_de_pares) para a√±adir o actualizar m√∫ltiples √≠tems a la vez. Si las claves ya existen, sus valores se actualizan; si no, se a√±aden los nuevos pares. [cite: 110, 121, 192]

3.3. Eliminando Pares Clave-Valor

Python ofrece varias formas de eliminar √≠tems:

  • del diccionario[clave]: Elimina el √≠tem. Lanza KeyError si la clave no existe. [cite: 184]
  • pop(clave, valor_por_defecto_opcional): Elimina la clave y devuelve su valor. Si la clave no existe, devuelve valor_por_defecto (o lanza KeyError si no se da). [cite: 110, 189]
  • popitem(): Elimina y devuelve un par (clave, valor). En Python 3.7+, es el √∫ltimo √≠tem insertado (LIFO). Lanza KeyError si est√° vac√≠o. [cite: 110, 190]
  • clear(): Elimina todos los √≠tems. [cite: 110, 191]
# Registro de mediciones de un sensor (F√≠sica/Ingenier√≠a)
mediciones_sensor = {
    "temperatura_C": 25.5, 
    "humedad_relativa_porc": 60.2, 
    "presion_hPa": 1012.5,
    "timestamp_ultimo": "2024-05-30 10:00:00"
}
print(f"Mediciones iniciales: {mediciones_sensor}")

# Eliminar 'presion_hPa' con del
if "presion_hPa" in mediciones_sensor: # Buena pr√°ctica verificar antes
    del mediciones_sensor["presion_hPa"]
print(f"Tras del: {mediciones_sensor}")

# Eliminar 'humedad_relativa_porc' con pop() y obtener su valor
humedad = mediciones_sensor.pop("humedad_relativa_porc", "No registrada")
print(f"Humedad eliminada: {humedad}. Mediciones: {mediciones_sensor}")

# Eliminar el √∫ltimo √≠tem (timestamp_ultimo en Python 3.7+)
if mediciones_sensor: # Verificar si no est√° vac√≠o
    clave_elim, valor_elim = mediciones_sensor.popitem()
    print(f"Eliminado con popitem(): '{clave_elim}': {valor_elim}")
print(f"Mediciones tras popitem(): {mediciones_sensor}")

# Vaciar todas las mediciones
mediciones_sensor.clear()
print(f"Mediciones finales: {mediciones_sensor}")

4. Vistas del Diccionario: keys(), values(), items()

Estos m√©todos devuelven objetos especiales llamados "vistas de diccionario". Una vista es din√°mica: si el diccionario cambia, la vista refleja esos cambios. Son iterables, lo que te permite usarlas en bucles for. [cite: 78, 79, 110, 185, 186, 187]

  • diccionario.keys() : Devuelve una vista de todas las claves.
  • diccionario.values() : Devuelve una vista de todos los valores.
  • diccionario.items() : Devuelve una vista de todos los pares clave-valor (como tuplas (clave, valor)).
# Ejemplo: Informaci√≥n nutricional de alimentos (Biolog√≠a/Nutrici√≥n)
info_nutricional_por_100g = {
    "Manzana": {"calorias": 52, "carbohidratos_g": 14, "fibra_g": 2.4},
    "Pollo (pechuga)": {"calorias": 165, "proteina_g": 31, "grasa_g": 3.6},
    "Lentejas (cocidas)": {"calorias": 116, "proteina_g": 9, "fibra_g": 7.9}
}

print("--- Alimentos Registrados (Claves) ---")
for alimento in info_nutricional_por_100g.keys():
    print(f"- {alimento}")

print("\n--- Datos Nutricionales (Valores) ---")
# Aqu√≠ los valores son otros diccionarios
for datos_completos_alimento in info_nutricional_por_100g.values():
    print(f"- Calor√≠as: {datos_completos_alimento.get('calorias', 'N/A')}, Prote√≠nas: {datos_completos_alimento.get('proteina_g', 'N/A')}g")

print("\n--- Ficha Nutricional Completa (Items) ---")
for alimento, detalles_nutricionales in info_nutricional_por_100g.items():
    print(f"\nAlimento: {alimento}")
    for nutriente, valor in detalles_nutricionales.items():
        print(f"  - {nutriente.replace('_', ' ').capitalize()}: {valor}")

# Convertir vistas a listas si necesitas una copia est√°tica
lista_alimentos = list(info_nutricional_por_100g.keys())
print(f"\nLista est√°tica de alimentos: {lista_alimentos}")

7. Diccionarios Modernos: Mejoras Clave

Python evoluciona, ¬°y sus diccionarios tambi√©n!

  • Orden de Inserci√≥n Garantizado (desde Python 3.7): ¬°Los diccionarios recuerdan el orden en que a√±adiste los elementos! [cite: 97, 213]
  • Operador de Uni√≥n | (desde Python 3.9): Combina dos diccionarios para crear uno nuevo. Si hay claves repetidas, prevalece el valor del diccionario de la derecha. [cite: 98, 199]
  • Operador de Actualizaci√≥n In-Place |= (desde Python 3.9): Actualiza un diccionario con los √≠tems de otro, modificando el diccionario original. [cite: 101, 200]
# Ejemplo de orden (Python 3.7+)

# Datos de un compuesto qu√≠mico
propiedades_basicas_comp = {"formula": "NaCl", "estado": "S√≥lido"}
propiedades_fisicas_comp = {"punto_fusion_C": 801, "densidad_g_cm3": 2.17, "estado": "S√≥lido Cristalino"}

8. Comprensi√≥n de Diccionarios (Forma Pyth√≥nica)

Al igual que las listas por comprensi√≥n, Python te permite crear diccionarios de una forma concisa y elegante usando la comprensi√≥n de diccionarios. La sintaxis es {clave_exp: valor_exp for item in iterable if condicion}. [cite: 73, 197] 

# Crear un diccionario de n√∫meros y sus cuadrados

cuadrados_dict = {n: n**2 for n in numeros}
