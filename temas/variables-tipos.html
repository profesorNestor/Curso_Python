1. ¬øQu√© son las Variables? Entendiendo los Contenedores de Datos

Imagina que est√°s en un laboratorio üß™. Tienes diferentes recipientes para guardar sustancias: uno para el agua, otro para el √°cido, uno para una muestra de suelo, etc. Cada recipiente tiene una etiqueta que te dice qu√© contiene. En programaci√≥n, una variable es muy similar: es como un contenedor con una etiqueta (su nombre) que guarda un valor o dato espec√≠fico.

Puedes pensar en una variable como un espacio reservado en la memoria de la computadora donde se almacena informaci√≥n. Lo m√°s importante es que el valor guardado en una variable puede cambiar durante la ejecuci√≥n de un programa, ¬°pero su nombre (la etiqueta) generalmente permanece igual! üì¶‚ú®

En Python, las variables son m√°s t√©cnicamente referencias o "punteros" a objetos que residen en la memoria. Esto significa que el nombre de la variable no contiene el dato en s√≠ mismo, sino la direcci√≥n de memoria donde se encuentra ese dato. Esta caracter√≠stica es parte de lo que hace a Python tan din√°mico y flexible.

üîç Analog√≠a Cient√≠fica: Piensa en una variable llamada concentracion_glucosa. En un momento, podr√≠a almacenar el valor 90 (mg/dL), representando la glucosa en ayunas de un paciente. Despu√©s de una comida, este valor podr√≠a actualizarse a 140 (mg/dL). La "etiqueta" concentracion_glucosa sigue siendo la misma, pero el dato que "apunta" ha cambiado.

2. Definici√≥n y Asignaci√≥n de Variables

Crear (o "declarar") una variable en Python es muy sencillo. Simplemente eliges un nombre para tu variable y usas el signo igual (=), conocido como el operador de asignaci√≥n, para darle un valor.

  3. Hace que nombre_de_la_variable apunte o referencie a la ubicaci√≥n en memoria de ese valor.

3. Estilos de Nomenclatura en Python (PEP 8)

Elegir buenos nombres para tus variables es crucial. No solo se trata de que el c√≥digo funcione, sino de que sea f√°cil de leer y entender. En Python, la gu√≠a de estilo oficial PEP 8 nos da recomendaciones claras. ¬°Vamos a ver las m√°s importantes!

Snake Case (snake_case)

Palabras separadas por guiones bajos, todo en min√∫sculas. Es la convenci√≥n est√°ndar y recomendada en Python para variables y nombres de funciones.

# Variables cient√≠ficas:
masa_particula_kg = 1.67e-27
velocidad_inicial_mps = 25.0
energia_cinetica_julios = 0.0

Camel Case (camelCase)

Primera palabra en min√∫scula, siguientes con may√∫scula inicial. No recomendado en Python para variables (m√°s com√∫n en JavaScript).

// Evitar en Python para variables:
masaParticula = 1.67e-27
velocidadInicial = 25.0

Pascal Case (PascalCase)

Cada palabra empieza con may√∫scula. Exclusivo para nombres de Clases en Python.

# Solo para Clases:
class CalculadoraFisica:
    pass
# No para variables: MasaObjeto = 5.0

All Caps (ALL_CAPS_SNAKE_CASE)

Todo en may√∫sculas, con guiones bajos. Usado para constantes.

# Constantes:
PI = 3.14159265
GRAVEDAD_TIERRA = 9.80665 # m/s¬≤

üëç En Resumen: Para la mayor√≠a de tus variables y funciones en Python, ¬°usa snake_case! Es lo m√°s "Pyth√≥nico" y facilitar√° la lectura. Reserva ALL_CAPS_SNAKE_CASE para constantes.

5. Tipado Din√°mico: La Flexibilidad de Python

Una caracter√≠stica distintiva de Python es que es un lenguaje de tipado din√°mico. Esto significa que no necesitas declarar expl√≠citamente el tipo de una variable antes de usarla. Python infiere autom√°ticamente el tipo de dato en el momento en que le asignas un valor.

Adem√°s, una misma variable puede "apuntar" a datos de diferentes tipos en diferentes momentos durante la ejecuci√≥n del programa.

print(f"Valor: {mi_variable}, Tipo: {type(mi_variable)}")

mi_variable = "Hola, cient√≠fico!" # Ahora mi_variable apunta a una cadena (str)
print(f"Valor: '{mi_variable}', Tipo: {type(mi_variable)}")

mi_variable = 3.14159   # Y ahora es un flotante (float)
print(f"Valor: {mi_variable}, Tipo: {type(mi_variable)}")

Esta flexibilidad es muy conveniente y hace que Python sea r√°pido para prototipar y aprender. Sin embargo, tambi√©n significa que debes tener cuidado y saber qu√© tipo de dato esperas en tus variables para evitar errores en tiempo de ejecuci√≥n.

ü§î Contraste con Tipado Est√°tico: En lenguajes de tipado est√°tico (como Java o C++), debes declarar el tipo de la variable cuando la creas (ej. int edad = 30;) y esa variable solo podr√° contener datos de ese tipo. Python lo hace m√°s f√°cil al principio, ¬°pero la responsabilidad de usar los tipos correctamente recae en ti!

6. Anotaciones de Tipo (Type Hints) - Una Gu√≠a Opcional

Aunque Python es de tipado din√°mico, a partir de Python 3.5, se introdujeron las anotaciones de tipo. Son una forma opcional de indicar qu√© tipo de dato se espera para una variable, un par√°metro de funci√≥n o el valor que devuelve una funci√≥n.

    """Calcula la energ√≠a cin√©tica. Espera masa y velocidad como float, devuelve float."""
    return 0.5 * masa * (velocidad ** 2)

energia: float = calcular_energia_cinetica(masa=10.0, velocidad=5.0)
print(f"Energ√≠a calculada: {energia} Joules")

Puntos Clave sobre Type Hints:

  • Opcionales: Python funciona perfectamente sin ellas.
  • No Verificadas por Python en Tiempo de Ejecuci√≥n: Por defecto, Python no lanza errores si los tipos no coinciden con las anotaciones.
  • Para Legibilidad y Herramientas: Ayudan a otros (y a tu yo futuro) a entender el c√≥digo. Son muy √∫tiles para editores de c√≥digo (autocompletado) y herramientas de an√°lisis est√°tico (como MyPy) que s√≠ pueden verificar los tipos.

üí° Recomendaci√≥n: Al principio, enf√≥cate en entender los tipos b√°sicos. A medida que tus proyectos se vuelvan m√°s complejos, especialmente si trabajas en equipo o desarrollas funciones y clases reutilizables, las anotaciones de tipo ser√°n una herramienta muy valiosa.

7. Conversi√≥n de Tipos Expl√≠cita (Type Casting)

A veces, necesitas convertir un valor de un tipo de dato a otro. Por ejemplo, la funci√≥n input() siempre devuelve el dato ingresado por el usuario como una cadena de texto (str), incluso si el usuario escribe un n√∫mero. Para realizar operaciones matem√°ticas con esa entrada, primero debes convertirla a un tipo num√©rico (int o float). Este proceso de conversi√≥n expl√≠cita se llama type casting.

Python ofrece funciones incorporadas para estas conversiones:

  • int(valor): Intenta convertir valor a un entero.
  • float(valor): Intenta convertir valor a un n√∫mero de punto flotante.
  • str(valor): Convierte valor a una cadena de texto.
  • bool(valor): Convierte valor a un booleano (la mayor√≠a de los valores son True; 0, 0.0, "", [], None, etc., son False).

Ejemplos de Conversi√≥n de Tipos:

print(f"En 10 a√±os tendr√°s: {edad_numero + 10} a√±os")

altura_texto = input("Ingresa tu altura en metros (ej: 1.75): ")
altura_flotante = float(altura_texto)   # Convertimos a float
print(f"El doble de tu altura es: {altura_flotante * 2} m")

# De n√∫mero a cadena
numero_experimento = 42
id_experimento = "EXP-" + str(numero_experimento) # Convertimos 42 a "42" para concatenar
print(id_experimento) # Salida: EXP-42

# De n√∫mero a booleano
cantidad_muestras = 0
hay_muestras = bool(cantidad_muestras) # False, porque cantidad_muestras es 0
print(f"¬øHay muestras?: {hay_muestras}")

lectura_sensor = 0.001
sensor_activo = bool(lectura_sensor)  # True, porque no es cero
print(f"¬øSensor activo?: {sensor_activo}")

‚ö†Ô∏è ¬°Cuidado con los ValueError! Si intentas una conversi√≥n inv√°lida, como int("hola") o float("tres.catorce"), Python generar√° un error de tipo ValueError porque no puede realizar esa conversi√≥n. Aprender√°s a manejar estos errores con bloques try-except en lecciones futuras.

9. El Valor Especial: None

En Python, None es una palabra reservada y un objeto √∫nico que representa la ausencia de valor o un valor nulo. Es de tipo NoneType.
