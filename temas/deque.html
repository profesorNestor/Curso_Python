Listas Deque en Python: Colas de Doble Extremo Eficientes ‚ÜîÔ∏è

Descubre las `deque`, una estructura de datos optimizada para a√±adir y quitar elementos r√°pidamente por ambos extremos.

1. ¬øQu√© es una `deque` y por qu√© usarla?

En Python, adem√°s de las listas est√°ndar [], existe una estructura de datos especializada muy √∫til llamada deque (pronunciada como "deck", la abreviatura de "double-ended queue" o "cola de doble extremo"). Las deques est√°n disponibles en el m√≥dulo incorporado collections.

Una deque es una secuencia similar a una lista, pero con una gran ventaja: est√° altamente optimizada para realizar operaciones de a√±adir (append) y quitar (pop) elementos desde ambos extremos (izquierda y derecha) de manera muy r√°pida y eficiente.

Principales Ventajas de deque sobre las Listas Est√°ndar:

  • Eficiencia en los Extremos: Mientras que en una lista normal, operaciones como lista.insert(0, valor) (a√±adir al principio) o lista.pop(0) (quitar del principio) son lentas (complejidad $O(N)$, donde N es el tama√±o de la lista, porque todos los dem√°s elementos deben desplazarse), en una deque, las operaciones equivalentes (appendleft(), popleft()) son muy r√°pidas (complejidad $O(1)$, tiempo constante).
  • Implementaci√≥n de Pilas y Colas: Son perfectas para implementar eficientemente estructuras de datos como pilas (LIFO - Last In, First Out) usando append() y pop(), y colas (FIFO - First In, First Out) usando append() y popleft().
  • Historial Limitado: Se pueden crear deques con una longitud m√°xima fija. Si se a√±aden elementos a una deque llena, los elementos del extremo opuesto se descartan autom√°ticamente, lo cual es ideal para mantener un historial de los √∫ltimos N √≠tems.

Para usar una deque, primero debes importarla desde el m√≥dulo collections. 

from collections import deque

# Crear una deque vac√≠a
cola_de_tareas = deque()

# Crear una deque a partir de una lista existente
datos_sensor_iniciales = [10.1, 10.3, 10.2]
buffer_sensor = deque(datos_sensor_iniciales)
print(f"Deque inicial: {buffer_sensor}")

# Crear una deque con una longitud m√°xima (maxlen)
# √ötil para historiales o buffers circulares
ultimas_lecturas = deque(maxlen=5) 
print(f"Deque con longitud m√°xima (vac√≠a): {ultimas_lecturas}")

üí° ¬øCu√°ndo elegir deque? Si tu aplicaci√≥n requiere a√±adir o quitar elementos frecuentemente del inicio de una secuencia, o si necesitas una cola o pila eficiente, ¬°deque es tu mejor opci√≥n! Para acceso aleatorio por √≠ndice en el medio de la secuencia, las listas normales siguen siendo adecuadas.

2. M√©todos Principales de las Deques

Las deques comparten muchos m√©todos con las listas, pero tienen algunos espec√≠ficos para operaciones eficientes en ambos extremos.

üìã Tabla Resumen de M√©todos Clave de deque:

M√©todo Descripci√≥n Breve Complejidad
append(x) A√±ade x al extremo derecho (final) de la deque. $O(1)$
appendleft(x) A√±ade x al extremo izquierdo (inicio) de la deque. $O(1)$
pop() Elimina y devuelve el elemento del extremo derecho. Lanza IndexError si est√° vac√≠a. $O(1)$
popleft() Elimina y devuelve el elemento del extremo izquierdo. Lanza IndexError si est√° vac√≠a. $O(1)$
extend(iterable) A√±ade los elementos de iterable al extremo derecho. $O(k)$ (k=longitud del iterable)
extendleft(iterable) A√±ade los elementos de iterable al extremo izquierdo. Los elementos se a√±aden en orden inverso al del iterable. $O(k)$
remove(value) Elimina la primera aparici√≥n de value. Lanza ValueError si no est√°. $O(N)$
clear() Elimina todos los elementos de la deque. $O(N)$
rotate(n=1) Rota la deque n pasos a la derecha. Si n es negativo, rota a la izquierda. $O(N)$
reverse() Invierte los elementos de la deque in-place. $O(N)$
count(x) Cuenta las apariciones de x. $O(N)$
index(x[, start[, stop]]) Devuelve el √≠ndice de la primera aparici√≥n de x. Lanza ValueError si no est√°. $O(N)$
maxlen (atributo) Tama√±o m√°ximo de la deque (si se especific√≥ al crearla), o None. $O(1)$

Exploremos algunos de estos m√©todos con ejemplos:

append(x) y appendleft(x)

A√±adir elementos a ambos extremos.

from collections import deque
# Simulaci√≥n de llegada de tareas a un procesador
tareas_pendientes = deque(["TareaC"])
tareas_pendientes.append("TareaD")     # A√±ade al final (derecha)
tareas_pendientes.appendleft("TareaB") # A√±ade al inicio (izquierda)
tareas_pendientes.appendleft("TareaA")
print(f"Cola de tareas: {tareas_pendientes}")
# Salida: deque(['TareaA', 'TareaB', 'TareaC', 'TareaD'])

pop() y popleft()

Eliminar y obtener elementos de ambos extremos.

# Procesando la cola de tareas (FIFO - First In, First Out)
primera_tarea = tareas_pendientes.popleft()
print(f"Procesando: {primera_tarea}") # Sale 'TareaA'
print(f"Tareas restantes: {tareas_pendientes}")

rotate(n)

Rota los elementos. n > 0 rota a la derecha, n < 0 rota a la izquierda.

# F√≠sica: Simular un cifrado C√©sar o un carrusel
secuencia_original = deque(['A', 'B', 'C', 'D', 'E'])
secuencia_original.rotate(2)  # Rota 2 posiciones a la derecha
print(f"Rotada +2: {secuencia_original}") 
# Salida: deque(['D', 'E', 'A', 'B', 'C'])

secuencia_original.rotate(-1) # Rota 1 posici√≥n a la izquierda
print(f"Rotada -1: {secuencia_original}")
# Salida: deque(['E', 'A', 'B', 'C', 'D'])

maxlen (Atributo y Comportamiento)

Mantiene un historial de tama√±o fijo. Al a√±adir a una deque llena, los elementos del extremo opuesto se descartan.

# Biolog√≠a: √öltimas N mediciones de pH en un cultivo celular
ultimos_ph = deque(maxlen=3)
ultimos_ph.append(7.4)
ultimos_ph.append(7.3)
ultimos_ph.append(7.5)
print(f"√öltimos pH: {ultimos_ph}") # deque([7.4, 7.3, 7.5], maxlen=3)

ultimos_ph.append(7.2) # A√±ade 7.2, y 7.4 (el m√°s antiguo) se va
print(f"Despu√©s de a√±adir 7.2: {ultimos_ph}") 
# Salida: deque([7.3, 7.5, 7.2], maxlen=3)

3. Aplicaciones Pr√°cticas de las Deques

La eficiencia de las deques en operaciones de extremo las hace ideales para varios escenarios:

Implementaci√≥n de Pilas (LIFO)

Una pila sigue el principio "√öltimo en Entrar, Primero en Salir". Usa append() para apilar (push) y pop() para desapilar.

# Matem√°ticas: Evaluar expresiones con par√©ntesis
pila_operaciones = deque()
pila_operaciones.append("Operaci√≥n1") # Push
pila_operaciones.append("Operaci√≥n2") # Push
print(f"Pila: {pila_operaciones}")
elemento_sacado = pila_operaciones.pop() # Pop "Operaci√≥n2"
print(f"Sacado: {elemento_sacado}, Pila ahora: {pila_operaciones}")

Implementaci√≥n de Colas (FIFO)

Una cola sigue el principio "Primero en Entrar, Primero en Salir". Usa append() para encolar y popleft() para desencolar.

# Simulaci√≥n de cola de impresi√≥n (Inform√°tica)
cola_impresion = deque()
cola_impresion.append("DocumentoA.pdf")
cola_impresion.append("ImagenB.jpg")
print(f"Cola: {cola_impresion}")
trabajo_actual = cola_impresion.popleft() # Sale "DocumentoA.pdf"
print(f"Imprimiendo: {trabajo_actual}, Cola ahora: {cola_impresion}")

Historial de "Deshacer" (Undo)

Mantener un historial limitado de acciones recientes (usando maxlen).

# Editor de texto simple
historial_acciones = deque(maxlen=5)
def realizar_accion(accion):
    historial_acciones.append(accion)
    print(f"Acci√≥n: {accion}, Historial: {list(historial_acciones)}")

realizar_accion("Escribir 'Hola'")

realizar_accion("Guardar")
realizar_accion("Insertar imagen")
realizar_accion("A√±adir tabla")
realizar_accion("Borrar p√°rrafo") # "Escribir 'Hola'" se descarta

Ventanas Deslizantes (Procesamiento de Se√±ales/Datos)

Analizar subconjuntos de datos de tama√±o fijo a medida que llegan nuevos datos (F√≠sica/Estad√≠stica).

# Simulaci√≥n de datos de un sensor de temperatura
datos_ventana = deque(maxlen=4)

        print(f"Ventana: {list(datos_ventana)}, Promedio: {promedio_ventana:.2f}¬∞C")
